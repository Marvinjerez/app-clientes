# Planificación de actividades - Desarrollo de Microservicio y Aplicación Móvil

## Fase 1 - Análisis y diseño
**Duración estimada:**  30 minutos

### Actividades 

- Analisis de los requerimientos.
- Definición de la arquitectura del sistema (microservicio + aplicación móvil).
- Diseño del modeo de bases de datos para la gestión de clientes.
- Definición de los endpoints de la API REST.

### Entregables

- Modelo de base de datos.
- Lista de endpoints REST.

## Fase 2 – Configuración del Entorno

**Duración estimada:** 1 hora

### Actividades

- Instalación y configuración del entorno de desarrollo.
- Creación del proyecto del microservicio en Go.
- Instalación de dependencias necesarias (router, driver de base de datos).
- Configuración de la conexión con la base de datos.

### Entregables

- Proyecto base funcional.
- Conexión a base de datos establecida.

## Fase 3 – Desarrollo del Microservicio

**Duración estimada:** 6 horas

### Actividades

- Implementación del modelo de datos para clientes.
- Desarrollo de los controladores del microservicio.
- Implementación de las rutas de la API REST.
- Implementación de operaciones CRUD:
  - Crear cliente
  - Listar clientes
  - Obtener cliente por ID
  - Actualizar cliente
  - Eliminar cliente
- Manejo de errores y validaciones.
- Pruebas iniciales de los endpoints.

### Entregables

- API REST funcional.
- Microservicio ejecutándose correctamente.

## Fase 4 – Pruebas del Microservicio

**Duración estimada:** 30 minutos

### Actividades

- Pruebas de endpoints utilizando herramientas como Postman o curl.
- Verificación de respuestas en formato JSON.
- Validación de manejo de errores y casos incorrectos.

### Entregables

- API validada y funcionando correctamente.

## Fase 5 – Desarrollo de la Aplicación Móvil

**Duración estimada:** 8 horas

### Actividades

- Creación del proyecto de aplicación móvil.
- Implementación de pantalla de listado de clientes.
- Implementación de formulario para registrar clientes.
- Implementación de actualización de clientes.
- Implementación de eliminación de clientes.
- Integración con el microservicio mediante consumo de la API REST.
- Manejo de respuestas JSON.

### Entregables

- Aplicación móvil funcional conectada al microservicio.

## Fase 6 – Integración y Pruebas Finales

**Duración estimada:** 1.5 horas

### Actividades

- Pruebas de integración entre la aplicación móvil y el microservicio.
- Verificación del flujo completo de creación, consulta, actualización y eliminación de clientes.
- Corrección de errores detectados.

### Entregables

- Sistema completamente funcional.

## Fase 7 – Documentación y Entrega

**Duración estimada:** 2 horas

### Actividades

- Elaboración del archivo README con instrucciones de ejecución.
- Organización del código fuente del microservicio.
- Organización del código de la aplicación móvil.
- Preparación del script de base de datos.

### Entregables

- Código fuente del microservicio.
- Código fuente de la aplicación móvil.
- Script de base de datos.
- Documento README con instrucciones de ejecución.
- Planificación de actividades.
